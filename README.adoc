== Overview

MetFont is an open-source, freely licensed color font encoding 541 WMO (World Meteorological Organization) and ICAO meteorological symbols as Unicode glyphs in the link:https://en.wikipedia.org/wiki/Private_Use_Areas#Private_Use_Area_(PUA)[Private Use Area] (PUA, U+E000–U+E7FF). Curated by the OGC MetOcean DWG from WMO and ICAO standards.

The glyphs are organized into 7 semantic blocks — one per WMO symbol category — each with room for future additions.

The font provides standardized weather symbols used in:

* METAR/TAF aviation reports
* SYNOP synoptic observations
* Marine weather bulletins
* Weather charts and maps

Website: https://www.metfont.org/[www.metfont.org]

== Unicode Block Allocation

The 541 glyphs are organized into 7 semantic blocks within the BMP Private Use Area, analogous to Unicode planes:

[cols="1,2,1,3"]
|===
| Block | PUA Range | Glyphs | Categories
| Sky & Cloud | U+E000–U+E0FF | 50 | CloudHigh (CH), CloudLow (CL), CloudMedium (CM), CloudGenus (C), TotalCloudCover (N)
| Present Weather | U+E100–U+E2FF | 270 | PresentWeather (ww), Auto Station (wawa), Additional (w1w1)
| Past Weather | U+E300–U+E3FF | 20 | PastWeather (W1W2), Auto Station (Wa1Wa2)
| State of Ground | U+E400–U+E4FF | 20 | StateOfGround (E), StateOfGround (E')
| Pressure | U+E500–U+E5FF | 14 | PressureTendency (a), PressureCentres
| Wind & Ocean | U+E600–U+E6FF | 107 | WindArrows (dd, ff), SwellDirection, ShipDirection
| Significant Weather | U+E700–U+E7FF | 60 | ICAO SigWx, Fronts (Ft)
|===

Each block has 256 or 512 codepoints allocated, with room for future WMO symbol additions (1,507 free codepoints total).

== Quick Start

=== Web Embedding

[source,css]
----
@font-face {
  font-family: 'MetFont';
  src: url('MetFont-glyf.woff2') format('woff2');
  font-weight: normal;
  font-style: normal;
}

.weather {
  font-family: 'MetFont', sans-serif;
}
----

=== CSS Codepoints

[source,css]
----
.rain::before              { content: '\E713'; }
.thunderstorm::before      { content: '\E71A'; }
.snow::before              { content: '\E719'; }
.fog::before               { content: '\E71F'; }
.drizzle::before           { content: '\E703'; }
.tropical-cyclone::before  { content: '\E71B'; }
.cloud-high::before        { content: '\E000'; }
.wind-calm::before         { content: '\E600'; }
----

<span class="weather">&#xE713;</span>  <!-- Rain -->

<span class="weather">&#xE71B;</span>  <!-- Tropical cyclone -->

== Building from Source

=== Prerequisites

* Python 3.11+
* Node.js 18+
* `pip install nanoemoji picosvg fonttools lxml`

=== Full Pipeline

[source,bash]
----
git clone --recurse-submodules https://github.com/MetFont/metfont.git
cd metfont

npm run prepare-sources    # Assign codepoints + clean upstream SVGs
npm run prepare-fonts      # Normalize SVGs for font compilation
npm run generate-fonts     # Build all font formats
----

=== Individual Steps

[source,bash]
----
# 1. Discover SVGs and assign PUA codepoints
python3 helpers/assign-codepoints.py
# Outputs: data/codepoint-map.json, data/icons.json, data/plane-info.json

# 2. Clean upstream SVGs (fix errors, convert text to paths)
python3 helpers/clean-sources.py
# Outputs: src/ directory (541 cleaned SVGs)

# 3. Normalize for font (stroke-to-fill, viewBox 0 0 72 72)
python3 helpers/normalize-svg.py \
  --input-dir src/ \
  --mapping data/codepoint-map.json \
  --output-dir color/svg/
# Outputs: color/svg/ directory (541 hexcode-named SVGs)

# 4. Build fonts via nanoemoji
npm run generate-fonts
# Outputs: font/ directory
----

=== Website

The `dev` and `build:site` scripts automatically regenerate site data and copy
fonts before starting, so the site always matches the latest font build.

[source,bash]
----
npm run dev            # Regenerate glyphs.json + copy fonts, then start VitePress dev server
npm run build:site     # Regenerate glyphs.json + copy fonts, then production build
----

If you've already rebuilt fonts or changed data files and just want to refresh
the site without restarting:

[source,bash]
----
npm run generate-glyphs   # Regenerate site/public/glyphs.json + copy plane-info.json
npm run copy-fonts        # Copy font/ files to site/public/
----

To iterate on the full pipeline (SVGs → fonts → site):

[source,bash]
----
npm run prepare-sources    # Step 1: Assign codepoints + clean SVGs
npm run prepare-fonts      # Step 2: Normalize for font
npm run generate-fonts     # Step 3: Build fonts → font/
npm run dev                # Step 4: Copy fonts + regenerate data + start dev server
----

== Understanding Private Use Area (PUA)

The Unicode standard reserves U+E000–U+F8FF as a
https://www.unicode.org/faq/private_use.html[Private Use Area] — a sandbox
where organizations can assign codepoints for internal use without conflicting
with official Unicode characters.

MetFont uses the subrange **U+E000–U+E7FF** (541 codepoints across 7
semantic blocks). These characters display correctly only when MetFont is
loaded — other fonts will render boxes or question marks.
